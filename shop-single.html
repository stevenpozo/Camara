Contexto

El contexto bajo el cual fue hecho este proyecto consta de los siguientes items.

  1. Item 1:

    Ubicación de cámara y proyección de región “prohibida”

    la zona segura se define utilizando marcadores ArUco. Primero, se detectan los marcadores ArUco en la imagen y se extraen las esquinas de cada uno de ellos. Si se detectan cuatro marcadores, se considera que forman un polígono que delimita la zona segura. Las esquinas de los marcadores se utilizan para construir un polígono en la imagen que representa esta zona. Este polígono se dibuja en la imagen y su color cambia dependiendo de si se detecta una persona dentro de la zona segura, activando una alerta si es necesario.

  2. Item 2:

    Calibración de cámara.

    Primero, se preparan los puntos de objeto 3D y se cargan imágenes de un tablero de ajedrez para detectar los puntos de las esquinas. Se utilizan estos puntos para calcular los parámetros de la cámara. El proceso incluye encontrar las esquinas del tablero en cada imagen, refinar estas posiciones, y luego usar estos puntos para calcular la matriz de la cámara y los coeficientes de distorsión. Finalmente, se aplica la corrección de distorsión a una imagen de prueba y se calcula el error medio de reproyección para evaluar la precisión de la calibración. Los parámetros calibrados se guardan en un archivo XML para su uso futuro.

    También existe la calibración de la cámara en C++.

  3. Item 3:

    Alarmar si persona pasa la región prohibida.

    La emisión de alerta con sonido se implementa mediante una función llamada `playBeep()`, que utiliza la función `Beep()` de la biblioteca de Windows para generar un pitido de 1000 Hz durante 500 milisegundos. Esta función se llama desde el código cuando se detecta que una persona está dentro de una zona prohibida. Al combinar la detección de personas y la verificación de su ubicación en la zona segura, se activa el pitido como una forma de notificación sonora para alertar sobre la intrusión en la zona restringida.
